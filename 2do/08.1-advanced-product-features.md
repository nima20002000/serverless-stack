## Database Schema Changes

### New Models

#### Category Model
```prisma
model Category {
  id          String    @id @default(uuid())
  name        String
  slug        String    @unique
  description String?   @db.Text
  parentId    String?
  isActive    Boolean   @default(true)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  parent   Category?  @relation("CategoryHierarchy", fields: [parentId], references: [id])
  children Category[] @relation("CategoryHierarchy")
  products Product[]

  @@map("categories")
}
```

#### Tag Model
```prisma
model Tag {
  id        String    @id @default(uuid())
  name      String    @unique
  slug      String    @unique
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  products Product[]

  @@map("tags")
}
```

#### ProductMedia Model
```prisma
enum MediaType {
  IMAGE
  VIDEO
}

model ProductMedia {
  id        String    @id @default(uuid())
  productId String
  variantId String?
  type      MediaType
  url       String
  alt       String?
  order     Int       @default(0)
  createdAt DateTime  @default(now())

  product Product        @relation(fields: [productId], references: [id], onDelete: Cascade)
  variant ProductVariant? @relation(fields: [variantId], references: [id], onDelete: Cascade)

  @@map("product_media")
}
```

#### ProductVariant Model
```prisma
model ProductVariant {
  id          String   @id @default(uuid())
  productId   String
  name        String   // e.g., "Red - Large"
  sku         String?  @unique
  color       String?
  size        String?
  material    String?
  priceAdjust Decimal  @default(0) @db.Decimal(10, 2)
  stock       Int      @default(0)
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  product Product        @relation(fields: [productId], references: [id], onDelete: Cascade)
  media   ProductMedia[]

  @@map("product_variants")
}
```

### Updated Product Model
```prisma
model Product {
  id          String   @id @default(uuid())
  name        String
  description String   @db.Text
  price       Decimal  @db.Decimal(10, 2)
  stock       Int      @default(0)
  categoryId  String?
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  category         Category?          @relation(fields: [categoryId], references: [id])
  tags             Tag[]
  media            ProductMedia[]
  variants         ProductVariant[]
  transactionItems TransactionItem[]

  @@map("products")
}
```

**Note**: Remove `images: String[]` field - replaced by `ProductMedia` model

## Technical Considerations

### File Upload Strategy
**Option 1: Local File System (Simpler)**
- Store in `public/uploads/`
- Direct URL access
- Pros: Simple, no external dependencies
- Cons: Not scalable, files lost on redeployment

**Option 2: Cloud Storage (Recommended for production)**
- Use Cloudinary, AWS S3, or similar
- Pros: Scalable, CDN, image optimization
- Cons: Additional cost, external dependency

**Decision for v1**: Use local file system for simplicity, design API to be storage-agnostic for easy migration

### Image Optimization
- Use Next.js `<Image>` component for automatic optimization
- Generate thumbnails for gallery
- Lazy load images below the fold
- WebP format with fallbacks

### Video Handling
- Support MP4 and WebM formats
- Limit video size (50MB max)
- Use HTML5 video player
- Consider video thumbnails/posters

### Variant Logic
- Base product has base price/stock
- Variants have price adjustments (+ or - from base)
- Variants can have independent stock
- If no variant selected, use base product values
- Out-of-stock variants disabled but visible

### Data Migration
- Existing products have `images: String[]` field
- Migration strategy:
  1. Create new tables
  2. Migrate existing image URLs to ProductMedia records
  3. Remove old `images` column after verification
