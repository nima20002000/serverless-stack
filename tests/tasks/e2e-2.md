## Guest Checkout Flow Analysis

**What happens in a guest checkout:**

```
User Action                  → Frontend                → Backend                    → Database
─────────────────────────────────────────────────────────────────────────────────────────────
1. Browse /products          → Fetch products          → GET /api/products          → products table
2. Click product             → Navigate                → N/A                        → N/A
3. Click "Add to Cart"       → Update Zustand store    → N/A (localStorage)         → N/A
4. View cart                 → Read from Zustand       → N/A                        → N/A
5. Click "Checkout"          → Navigate to /checkout   → N/A                        → N/A
6. Fill phone: 09123456789   → Form state              → N/A                        → N/A
7. Click "Send OTP"          → POST /api/auth/send-otp → OTP service creates code   → otp_codes table
8. SMS sent (MOCKED in E2E)  → N/A                     → Kavenegar API (SKIP)       → N/A
9. Get OTP from DB           → N/A                     → SELECT from otp_codes      → otp_codes table
10. Enter OTP + verify       → POST /api/auth/verify-otp→ Update user as verified   → users table
11. Fill address + submit    → POST /api/transactions/create → Create transaction  → transactions table
12. Redirect to Zarinpal     → window.location         → Zarinpal redirect          → (MOCKED)
13. Mock payment success     → Route interception      → N/A                        → N/A
14. Callback verify          → GET /api/transactions/verify → Update tx status     → transactions table
15. Show success page        → Navigate to /success    → N/A                        → N/A
16. Verify order in DB       → N/A                     → SELECT from transactions   → transactions table
17. Verify stock decreased   → N/A                     → SELECT from products       → products table
```
