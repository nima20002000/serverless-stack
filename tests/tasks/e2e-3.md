## User Registration Flow Analysis

**Email Registration Flow:**

```
1. Navigate to /register
2. Fill email + password + name
3. Click "Register"
4. System sends OTP to email (MOCKED - use Ethereal)
5. Retrieve OTP from database (otp_codes table)
6. Enter OTP in verification modal
7. User created in database (users table)
8. NextAuth session created (JWT + cookie)
9. Redirect to /profile or /
10. Verify user can access protected routes
```

**Phone Registration Flow:**

```
1. Navigate to /register
2. Fill phone + name (no password)
3. Click "Register"
4. System sends OTP via SMS (MOCKED)
5. Retrieve OTP from database
6. Enter OTP
7. User created with isVerified=true
8. NextAuth session created
9. Redirect to profile
```
